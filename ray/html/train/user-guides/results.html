检查训练结果#

调用 Trainer.fit() 的返回结果是 一个 Result 对象。

Result 对象包含以下信息：

  • 最后报告的指标（例如损失）

  • 最后报告的检查点（加载模型）

  • 如果发生任何错误，则显示错误消息

查看指标#

您可以从 Result 对象检索报告给 Ray Train 的指标。

常见指标包括训练或验证损失，或预测准确性。

从 Result 对象检索到的指标与 您作为 在您的训练函数中 的参数 传递给 train.report 的指标相对应，。

最近报告的指标#

使用 Result.metrics 检索 最新报告的指标。

result = trainer.fit()

print("Observed metrics:", result.metrics)

所有报告指标的数据#

使用 :attr:`Result.metrics_dataframe <ray.train.Result.metrics_dataframe>`检索所有报告指标的 pandas DataFrame。

df = result.metrics_dataframe
print("Minimum loss", min(df["loss"]))

检索检查点#

您可以从 Result 对象检索报告给 Ray Train 的检查点。

Checkpoints 包含恢复训练状态所需的所有信息。、 这通常包括训练后的模型。

您可以将检查点用于常见的下游任务，例如 使用 Ray Data 进行离线批量推理, 或 使用 Ray Serve 进行在线模型服务。

Result 对象检索到的检查点与您 在训练函数中 作为参数 传递给 train.report 的检查点相对应。

最后保存的检查点#

使用 :attr:`Result.checkpoint <ray.train.Result.checkpoint>`检索最后一个检查点。

print("Last checkpoint:", result.checkpoint)

with result.checkpoint.as_directory() as tmpdir:
    # Load model from directory
    ...

其他检查点#

有时，您希望访问较早的检查点。例如，如果 由于过度拟合导致损失在更多训练后增加， 您可能希望检索损失最低的检查点。

您可以使用以下方式 Result.best_checkpoints 检索 所有可用检查点及其指标的列表

# Print available checkpoints
for checkpoint, metrics in result.best_checkpoints:
    print("Loss", metrics["loss"], "checkpoint", checkpoint)

# Get checkpoint with minimal loss
best_checkpoint = min(result.best_checkpoints, key=lambda bc: bc[1]["loss"])[0]

with best_checkpoint.as_directory() as tmpdir:
    # Load model from directory
    ...

访问存储位置#

如果您稍后需要检索结果，可以使用 Result.path 获取训练运行的存储位置。

此路径将对应于您在 RunConfig 中 配置的 storage_path 。 它将是该路径内的（嵌套）子目录，通常为 TrainerName_date-string/TrainerName_id_00000_0_... 的形式。

result_path = result.path
print("Results location", result_path)

你可以使用 Result.from_path 恢复结果：

from ray.train import Result

restored_result = Result.from_path(result_path)
print("Restored loss", result.metrics["loss"])

查看错误#

如果训练期间发生错误， Result.error 将会设置并包含引发的异常。

if result.error:
    assert isinstance(result.error, Exception)

    print("Got exception:", result.error)

在持久存储中查找结果#

有训练结果（包括报告的指标、检查点和错误文件）都存储 在配置的 持久化存储 中。

参考 我们的持久化存储指南，为您的 训练运行位置配置。
